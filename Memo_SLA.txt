## Memo Generation SLA & Escalation Procedures

Date:27-06-2026

---

# 1. Service Level Agreement (SLA)

The Memo Generation Service Level Agreement (SLA) defines the expected performance, reliability, and quality standards for AI-generated underwriting memos.

| SLA Metric                   | Target                                 |
| ---------------------------- | -------------------------------------- |
| Memo Generation Time         | Within 30 seconds after model decision |
| p95 Latency                  | Less than 30 seconds                   |
| p99 Latency                  | Less than 60 seconds                   |
| Recommendation Accuracy      | 100%                                   |
| Memo Generation Success Rate | 99.5% Uptime                           |
| Memo Completeness            | 100% Required Sections Present         |
| System Availability          | 99.5%                                  |

---

# 2. Memo Quality Standards

Every generated memo must include the following mandatory sections:

* Applicant Profile
* Risk Assessment
* Credit History
* Repayment Capacity
* Risk Factors
* Final Recommendation

Quality objectives:

* No hallucinated information
* Consistent recommendation
* Clear reasoning
* Complete structured response
* Professional language
* JSON format validation

---

# 3. Escalation Procedures

Automatic escalation rules are applied whenever the generated memo does not satisfy predefined quality or operational thresholds.

| Trigger Condition               | Escalation Action                                   | Responsible Team |
| ------------------------------- | --------------------------------------------------- | ---------------- |
| Memo Quality Score < 2          | Escalate for Manual Credit Analyst Review           | Credit Analyst   |
| Confidence Score < 50%          | Escalate to Senior Credit Expert                    | Senior Analyst   |
| Memo Generation Failure         | Escalate to Project Manager and Engineering Team    | PM / Engineering |
| Generation Latency > 60 Seconds | Log Performance Incident and Investigate Bottleneck | Engineering Team |
| Missing Mandatory Sections      | Manual Review Required                              | Credit Analyst   |
| Invalid Recommendation          | Manual Verification                                 | Senior Analyst   |

---

# 4. Escalation Dashboard Design

The escalation dashboard provides real-time monitoring of memo generation quality and SLA compliance.

### Dashboard Metrics

* Total Memos Generated
* Successful Memo Generations
* Failed Memo Generations
* Total Escalations
* Escalation Percentage
* Average Latency
* p95 Latency
* p99 Latency
* Recommendation Accuracy
* SLA Compliance Percentage

### Escalation Reasons

* Low Memo Quality
* Low Confidence Score
* Memo Generation Failure
* High Response Latency
* Missing Required Sections
* Recommendation Validation Failure

### Alerts

Generate system alert when:

* Escalations exceed 10% of generated memos
* Memo generation success rate drops below SLA target
* Latency exceeds p99 threshold
* API failures detected

---

# 5. Analyst Review Workflow

Workflow for escalated memo generation:

1. AI generates underwriting memo.
2. Automatic validation checks quality score, confidence score, latency, and completeness.
3. If no issues are detected, the memo is approved.
4. If any escalation rule is triggered:

   * Assign memo to Credit Analyst.
   * Analyst reviews memo manually.
   * Decision options:

     * Accept Memo
     * Reject Memo
     * Modify Memo
5. Analyst feedback is recorded.
6. Feedback is used to improve future memo generation quality.
